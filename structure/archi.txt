PROJECT OVERVIEW
================================================================================
Build a complete sentiment analysis system for Myanmar (Burmese) text capable of 
classifying social media comments, reviews, and user-generated content into 
Positive, Negative, and Neutral categories.

Time estimate: 4-6 weeks full-time or 2-3 months part-time
Difficulty: Advanced (⭐⭐⭐⭐)

PHASE 1: DATA COLLECTION & PREPROCESSING
================================================================================

Existing Datasets Available:

| Dataset Source                    | Type              | Size   | Access              |
|-----------------------------------|-------------------|--------|---------------------|
| kalixlouiis/burmese-sentiment     | Social media      | ~10k   | Hugging Face        |
| chuuhtetnaing/myanmar-sentiment   | Facebook comments | ~5k    | Hugging Face        |
| Custom collection                 | Various           | Variable| Scraping API        |

Key Preprocessing Steps:
────────────────────────────────────────────────────────────────────────────────
1. Zawgyi to Unicode conversion (critical - many Myanmar texts use Zawgyi)
2. Text cleaning (remove URLs, emojis, special characters)
3. Syllable tokenization using SimboloSiamese library
4. Train/Validation/Test split: 70%/15%/15%


PHASE 2: MODEL DEVELOPMENT
================================================================================

Base Model Comparison:

| Model              | Pros                              | Cons                    | Performance |
|--------------------|-----------------------------------|-------------------------|-------------|
| XLM-RoBERTa-base   | Multilingual, strong transfer     | Larger (550MB)          | 84.7% F1    |
| mBERT              | Supports 104 languages            | Older architecture      | Comparable  |
| BiLSTM+Attention   | Lightweight, interpretable        | Lower accuracy          | 85-86%      |

Recommended Hyperparameters (Proven Working Values):

| Parameter              | Value     | Why                                   |
|------------------------|-----------|---------------------------------------|
| Learning rate          | 2e-5      | Standard for fine-tuning              |
| Batch size             | 8         | Small dataset, prevents overfitting   |
| Epochs                 | 3         | More epochs cause overfitting         |
| Max sequence length    | 128       | Myanmar sentences typically short     |
| Optimizer              | AdamW     | Default for transformers              |
| Weight decay           | 0.01      | Regularization                        |
| Warmup steps           | 500       | Stable training start                 |
